# UI Redesign 设计规范

## 1. 设计目标

本次重构在原有蓝白清新基调上，升级为更轻盈、明亮的天蓝色知识工作台视觉。整体风格关键词：

- 清新
- 专业
- 精致
- 轻量
- 高可读性

## 2. 色彩系统

### 主色调

- `--primary-soft: #dff3ff`
- `--primary: #36a2eb`
- `--primary-deep: #4dabf7`
- `--primary-dark: #1786d0`
- `--primary-ink: #0a75bb`

用途：

- 主按钮、导航高亮、关键标签、预览切换
- 状态强调和界面点睛色

### 中性色

- `--bg: #eef7ff`
- `--bg-accent: #f7fbff`
- `--panel: rgba(255, 255, 255, 0.9)`
- `--panel-strong: #ffffff`
- `--panel-soft: #f6fbff`
- `--text: #16324f`
- `--text-strong: #0b2239`
- `--muted: #6b8298`
- `--muted-strong: #54708a`
- `--border: rgba(128, 177, 210, 0.28)`
- `--border-strong: rgba(104, 168, 218, 0.42)`

设计意图：

- 用更干净的浅蓝灰替代沉重深灰
- 提升背景明度，保留专业感

### 功能色

- `--ok: #22a46d`
- `--warn: #d68b1d`
- `--danger: #d15f6e`

规则：

- 成功用偏青绿，避免和主天蓝冲突
- 警告改为柔和琥珀色
- 错误采用偏粉红红色，减少刺眼程度

## 3. 阴影与深度

- `--shadow: 0 20px 60px rgba(31, 79, 121, 0.08)`
- `--shadow-soft: 0 10px 24px rgba(42, 96, 143, 0.08)`

规则：

- 大容器使用 `--shadow`
- 普通卡片使用 `--shadow-soft`
- 通过浅边框 + 柔阴影而不是重投影塑造层次

## 4. 间距系统

- `4px`
- `8px`
- `12px`
- `16px`
- `24px`
- `32px`
- `40px`
- `48px`

使用策略：

- 组件内部优先使用 `8 / 12 / 16`
- 模块与模块之间优先使用 `24 / 32`
- 页面级留白优先使用 `40 / 48`

## 5. 圆角系统

- `--radius-xs: 10px`
- `--radius-sm: 12px`
- `--radius-md: 16px`
- `--radius-lg: 20px`
- `--radius-xl: 28px`

规则：

- 按钮、输入框、标签：`12px`
- 卡片、列表项：`16px`
- Hero、主面板：`20px` 或 `28px`

## 6. 版式系统

### 导航

- 旧版深色厚重侧栏改为浅色玻璃感侧栏
- 侧栏宽度从 `260px` 收敛为 `232px`
- 强调更轻、更透、更现代

### 内容区

- 大屏默认更宽松留白
- Hero 区升级为带渐变高光的主视觉头图
- 面板边界更清晰但更轻

## 7. 响应式断点

- `>= 1200px`
  大桌面，增加页面留白并放大 Hero 布局
- `<= 1199px`
  平板横屏与普通桌面过渡，统计卡片变两列
- `<= 999px`
  侧栏上移为顶部模块，多栏改为单栏
- `<= 767px`
  移动端，按钮全宽、卡片单列、Hero 更紧凑

## 8. 组件规则

### Hero

- 用于页面一级信息和系统状态
- 带背景渐变与柔和装饰光斑

### Panel

- 页面主内容容器
- 使用浅渐变背景 + 柔边框 + 轻阴影

### Stats Card

- 通过伪元素增加微装饰和识别符号
- 强调数字信息的视觉记忆点

### 按钮

- 主按钮：天蓝色填充
- 次按钮：绿色操作型按钮
- Hover 只做轻微上浮和色阶变化

### 表单

- 焦点态统一蓝色描边 + 轻外发光
- 提高输入区域可识别度

### 表格

- 用浅蓝灰表头和更细分隔线提高扫描效率
- Hover 行高亮，用轻蓝背景提示交互

### 预览组件

- 支持渲染预览和原始文本切换
- 顶部显示文件路径、文件类型、大小信息
